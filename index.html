Assets

各资产的定位

LAMD-DATASETS

UAIV 生态的首次公开亮相，展示数据集的多样性和任务覆盖面。当前只公开约 10%，用于社区预览数据结构、标注格式和任务类型。

变化检测 生态 图像修复 ReID 城市场景

UAIV-Real-CityGov-v1

旗舰城市治理数据集，围绕 4,796 张低空图像，覆盖事件理解、目标检测、场景识别、城市功能、环境状态和 OCR 六大任务模块。

Scientific Data 4,796 images 6 tasks

UAIV-Labeler

低空原生半自动标注平台，强调飞行批次管理、多任务联合标注、模型辅助和 AI-Ready 导出。

Detection Segmentation OCR Event QA

UAIV-Foundry

低空数据生产操作系统，让数据构建可复现、可审计。包含 Pipeline、QA Runner、Dashboard、Dataset Card Generator 和 Benchmark Runner。

Pipeline QA Dashboard Benchmark

Relationship

三层关系

UAIV 数据基座的逻辑不是“把一堆链接放在一起”，而是把数据集、标注平台和 Foundry 组织成一个可持续更新的生产系统。

层级 角色 作用
Data Product UAIV-Real / LAMD-DATASETS 对外发布的数据资产和 benchmark 入口
Human Workbench UAIV-Labeler 人工标注、复核和 AI 辅助标注的生产台
Data OS UAIV-Foundry 把导入、标注、QA、发布和评测连接成闭环
Future Evaluation Layer UAIV-Benchmark / UAIV-QA-Benchmark 将来统一评测与方法比较的标准化基准层

Roadmap

后续路线图

短期

继续收尾 UAIV-Real-CityGov-v1 的标注和 QA；同步让 UAIV-Foundry 收敛成稳定可展示的发布形态。

中期

把 UAIV-Benchmark 和 UAIV-QA-Benchmark 作为未来统一评测层，逐步补齐 AI-Ready 数据格式与 baseline 协议。

长期

把 UAIV 数据基座做成持续更新的公开入口，让数据集、标注平台和 Foundry 都可以被外部直接访问和引用。

访问策略

已发布资产直接跳转，收尾中资产保留预览或源码入口，规划中资产仅展示状态与后续方向。
